# 台股買賣收支及盈絀管制系統 (Taiwan Stock Trading Control SPA)

一個基於 **Vue 3 (CDN)**、**Tailwind CSS (CDN)** 和 **Chart.js (CDN)** 打造的輕量單頁 Web 應用程式 (SPA)。
本系統完全採用**台灣繁體中文界面**與**台股習慣用語**（如：張數、折讓、盈絀），並嚴格遵循台灣證券交易所與稅務規範進行手續費、證交稅與已實現損益（加權平均成本法）的核算。

---

## 🚀 快速開始

本系統無須安裝任何後端環境，完全開箱即用。請選擇以下任一方式開啟：

- **方法一：直接點擊開啟 (最簡單)**
  - 在您的電腦中，直接用瀏覽器雙擊開啟專案目錄下的 `index.html`。
- **方法二：透過本機 Python 伺服器 (推薦)**
  - 在專案目錄開啟終端機 (Terminal)。
  - 執行指令：`python -m http.server 8080`
  - 在瀏覽器輸入：`http://localhost:8080`

## 💡 核心功能說明

本系統提供豐富的台股記帳與損益追蹤功能：

*   **總資產儀表板**：即時統計累積總投入成本、已實現損益、配息收入與總淨盈絀，並搭配台股「紅漲綠跌」配色呈現。
*   **即時試算表單**：輸入價格、張數、折讓即時試算交易金額，內建熱門台股快捷鍵與手續費折讓快速按鈕。
*   **動態損益報表**：提供【每月】與【每年】長條圖報表，視覺化呈現各週期的總收支與淨盈絀。
*   **本地資料儲存**：自動利用瀏覽器 LocalStorage 儲存您的交易紀錄，網頁重整資料不遺失。

## 📐 台股核心計算邏輯

### 1. 手續費與證交稅公式

*   **股數** = 張數 $\times$ 1,000
*   **原始手續費** = 價格 $\times$ 股數 $\times$ 0.001425
*   **折讓後手續費**：
    - 若張數 $\ge 1$：$\max(20, \lceil \text{原始手續費} \times \text{折讓比例} \rceil)$ *(手續費採無條件進位。若折讓後低於 20 元，按券商最低基本消費 20 元計收)*
    - 若張數 $< 1$ (零股)：$\lceil \text{原始手續費} \times \text{折讓比例} \rceil$ *(手續費採無條件進位，但不受最低基本消費 20 元限制，直接按折讓折扣計收)*
*   **證券交易稅** (僅賣出時課徵) = $\lfloor \text{價格} \times \text{股數} \times 0.003 \rfloor$  
    *(證交稅採無條件捨去至個位數)*
*   **收支總額**：
    - **買入支出** = $(\text{價格} \times \text{股數}) + \text{折讓後手續費}$
    - **賣出收入** = $(\text{價格} \times \text{股數}) - \text{折讓後手續費} - \text{證券交易稅}$
    - **配息收入** = $\text{每股配息} \times \text{股數}$

### 2. 已實現損益計算（加權平均成本法）

系統將歷史紀錄按「交易日期由舊到新」排序，動態計算持股庫存與均價：

-   **買入交易**：
    - 更新庫存股數與累計成本。
    - 重新計算每股加權平均持有成本：
      $$\text{加權平均成本} = \frac{\text{累計持有總成本}}{\text{累計持有總股數}}$$
-   **配息交易**：
    - 直接認列為單筆配息盈絀，**不扣減**持股庫存成本（符合一般個人所得記帳與申報習慣）。
-   **配股交易**：
    - 僅增加該股票的持股庫存股數，**不增加**庫存總成本（總收支為 0 元）。
    - 這會稀釋並降低每股平均成本，配股當下不認列已實現損益（盈絀顯示為 `-`）。
-   **賣出交易**：
    - 依據賣出前的加權平均每股成本計算對應買入成本：
      $$\text{對應買入成本} = \text{加權平均成本} \times \text{賣出股數}$$
    - 計算該筆賣出的已實現損益：
      $$\text{單筆盈絀} = \text{賣出總收入} - \text{對應買入成本}$$
    - 扣減對應的庫存股數與庫存成本。
-   **防禦防錯機制**：
    - 若記帳順序或手誤導致賣出股數大於目前庫存（超賣），超出庫存部分的成本將以「賣出價格」估算（即超出部分的損益計為 0），以防庫存總成本產生異常負數。

### 3. 輸入空格智慧防錯 (`getStockKey`)

為避免使用者記帳時輸入的代號或名稱空格不一致（例如 `2330 台積電` 與 `2330台積電`）被判定為不同股票，系統在核心比對時會自動：
1. 移除字串中所有的空格與括號。
2. 自動轉為英文字母大寫。
3. 優先提取開頭的 4 到 6 位純數字代號作為唯一配對鍵值。
4. 確保只要股票代號相同，加權成本便能百分之百正確累計。

# 台股買賣收支及盈絀管制系統 (Taiwan Stock Trading Control SPA)

一個基於 Vue 3 (CDN)、Tailwind CSS (CDN) 和 Chart.js (CDN) 的單頁 Web 應用程式 (SPA)。本系統完全採用台灣繁體中文界面與台灣股市習慣用語（如：張數、折讓、盈絀），並嚴格遵循台灣證券交易所與稅務規範進行手續費、證券交易稅與已實現損益（加權平均成本法）的核算。

---

## 🚀 快速開始

本系統為單網頁應用程式 (SPA)，完全開箱即用，無須複雜的環境建置：

### 方法一：直接點擊開啟
1. 在您的電腦中，直接使用瀏覽器開啟本專案根目錄的 `index.html`。

### 方法二：透過本機 HTTP 伺服器執行 (推薦，可正常載入部分瀏覽器之快取功能)
1. 在專案根目錄開啟終端機 (Terminal)。
2. 執行以下 Python 指令啟動輕量伺服器：
   ```bash
   python -m http.server 8000
   ```
3. 在瀏覽器中輸入以下網址即可使用：
   ```
   http://localhost:8000
   ```

## 💡 核心功能

1. **總資產儀表板**
   * **累積總投入成本**：累計所有買入交易的總支出金額（含手續費）。
   * **累積已實現損益**：已平倉賣出持股的損益累計。
   * **累積配息收入**：累計獲得的現金股利總額。
   * **總淨盈絀**：累積已實現損益與累積配息收入的加總，並即時估算投資報酬率 (ROI)。
   * **台股紅綠配色**：獲利（正數）以**紅色**呈現；虧損（負數）以**綠色**呈現。

2. **交易資料即時試算與輸入表單**
   * 當您在左側表單輸入「每股價格」與「張數」時，表單下方會**即時呈現試算預覽**（含股數、原始手續費、折讓後手續費、預估證交稅與總收支金額），讓您在送出前確認數值。
   * 提供常用熱門台股快捷鍵（如 `2330 台積電`、`2454 聯發科`、`2317 鴻海` 等），一鍵快速填入。
   * 手續費折讓預設為 6 折，並提供無折讓、2.8 折、1 折等快速按鈕，同時支援 0 ~ 1 之間任意數值的自訂比例。

3. **每月與每年盈絀報表**
   * 可靈活切換【每月報表】與【每年報表】模式。
   * 整合 Chart.js 動態渲染長條圖（Bar Chart），獲利月份/年度呈現紅色直條，虧損則呈現綠色直條。
   * 自動彙整各週期的總買入、總賣出、總配息與該週期淨盈絀。

4. **歷史交易紀錄與管理**
   * 支援股票代號/名稱關鍵字模糊搜尋、交易類型（買入、賣出、配息）分選過濾，並提供列表分頁功能。
   * 支援單筆交易紀錄的「編輯」與「刪除」功能。
   * 內建 LocalStorage 本地持久化儲存，重新整理網頁資料亦不會消失。
   * 提供「一鍵載入範例資料」按鈕，快速體驗系統全貌。
   * 提供「備份匯出 (JSON)」與「還原匯入 (JSON)」功能，方便資料轉移或手動備份。

## 📐 台股核心計算邏輯與規範

### 1. 手續費與證交稅公式
* **股數** = 張數 $\times$ 1,000
* **原始手續費** = 價格 $\times$ 股數 $\times$ 0.001425
* **折讓後手續費** = $\max(20, \lceil \text{原始手續費} \times \text{折讓比例} \rceil)$  
  *(計算金額採無條件進位至整數。若折讓後不足 20 元，按券商最低基本消 20 元計算)*
* **證券交易稅** (僅在「賣出」交易課徵) = $\lfloor \text{賣出價格} \times \text{股數} \times 0.003 \rfloor$  
  *(計算金額採無條件捨去至整數)*
* **買入總支出** = $(\text{價格} \times \text{股數}) + \text{折讓後手續費}$
* **賣出總收入** = $(\text{價格} \times \text{股數}) - \text{折讓後手續費} - \text{證券交易稅}$
* **配息總收入** = $\text{每股配息} \times \text{股數}$

### 2. 已實現損益（加權平均成本法）
系統會自動將所有歷史交易按照 **「交易日期由舊到新 $\to$ 同一天時買入優先 $\to$ 配息次之 $\to$ 賣出最後」** 的規則排序，並動態累計庫存：
* **買入**時：
  * 更新該檔股票庫存總股數與累計成本。
  * 算出最新的庫存加權平均每股成本：
    $$\text{加權平均成本} = \frac{\text{累計成本}}{\text{累計股數}}$$
* **配息**時：
  * 配息收入直接認列為單筆盈絀，不扣減持股庫存成本（符合一般個人記帳與報稅申報習慣）。
* **賣出**時：
  * 依據該檔股票賣出前的「加權平均每股成本」，算出對應的買入成本：
    $$\text{對應買入總成本} = \text{加權平均每股成本} \times \text{賣出股數}$$
  * 計算該筆賣出的已實現損益（單筆盈絀）：
    $$\text{單筆盈絀} = \text{賣出總收入} - \text{對應買入總成本}$$
  * 扣減該股票對應的庫存股數與庫存成本。
  * **防禦防錯機制**：若使用者因為記帳順序或手誤，導致賣出股數大於目前庫存股數，超出庫存部分的成本將以「賣出價格」估算（即超出部分的損益計為 0），以防庫存總成本產生異常的負數。

### 3. 輸入空格不一致的智慧防錯 (getStockKey)
實務上記帳時，使用者輸入的股票代號與名稱可能會有空格不一致的情況（例如，買入時輸入 `2344 華邦電`，賣出時輸入 `2344華邦電`）。
為了避免系統將其判定為兩檔不同股票而導致加權成本計算錯誤，系統在計算核心使用 `getStockKey` 函數進行標準化比對：
* 自動移除字串中所有的空格。
* 自動轉為大寫字母。
* 優先提取開頭的 4 到 6 位純數字代號（如 `2344`）做為匹配的唯一鍵值。
* 這能百分之百確保即使空格、括號或名稱有細微差異，只要股票代號相同，其庫存與損益加權平均皆能獲得正確的累計計算。
